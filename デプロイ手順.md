## あと2分：Vercelで公開するだけ

**[→ Vercelでインポートする（クリック）](https://vercel.com/new/clone?repository-url=https://github.com/Yp465-ck/tesou-kantei-tool)**

### 3. 環境変数を追加

- **Environment Variables** の **Key** に `OPENAI_API_KEY`
- **Value** に、`.env` ファイルの中の長いキー（`sk-proj-` で始まる部分）を貼り付け
- **Add** をクリック

### 4. Deploy をクリック

1〜2分待つと完了し、**Visit** ボタンから公開URLが開きます。

## 新規で1からデプロイする場合（古い反映のままになるとき）

古いキャッシュや前回のデプロイ設定が残っていると、更新が反映されないことがあります。その場合は、既存プロジェクトを削除して新規デプロイしてください。

### ステップ1: 既存プロジェクトを削除

1. [vercel.com](https://vercel.com) にログイン
2. 削除したいプロジェクト（tesou-shindan や tesou-kantei など）をクリック
3. 上メニューの **Settings** をクリック
4. 左メニュー一番下までスクロール → **Delete Project** をクリック
5. プロジェクト名を入力して確認し、**Delete** をクリック

### ステップ2: 新規プロジェクトとしてデプロイ

**[→ 新規インポート（クリック）](https://vercel.com/new/clone?repository-url=https://github.com/Yp465-ck/tesou-kantei-tool)**

または:

1. ダッシュボードで **Add New** → **Project**
2. **Import Git Repository** で **tesou-kantei-tool**（`Yp465-ck/tesou-kantei-tool`）を選択
3. **Configure Project** で:
   - **Project Name**: `tesou-shindan`（お好みの名前でOK）
   - **Environment Variables**: 空でOK
4. **Deploy** をクリック
5. 1〜2分待つと、新しいURL（例: `https://tesou-shindan.vercel.app`）で公開されます

## トラブルシューティング

### 「API key not configured」と出る場合

環境変数が設定されていません。次の手順で追加してください。

1. [vercel.com](https://vercel.com) にログイン
2. **tesou-kantei** プロジェクトをクリック
3. 上メニューの **Settings** をクリック
4. 左メニューの **Environment Variables** をクリック
5. **Name** に `OPENAI_API_KEY` を入力
6. **Value** に、`.env` ファイルに書いてあるAPIキー（`sk-proj-`で始まる長い文字列）を貼り付け
7. **Save** をクリック

### 環境変数を追加したあと：再デプロイのやり方

1. プロジェクトのトップ画面（**tesou-kantei**）を開く
2. 上の **Deployments** タブをクリック
3. 一番上のデプロイ（最新）の右端の **⋯**（3点リーダー）をクリック
4. **Redeploy** を選ぶ
5. 確認画面で **Redeploy** をクリック
6. 1〜2分待つ
7. 完了したら、もう一度URLを開いて確認する
